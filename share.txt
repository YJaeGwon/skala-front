## 1. 프로젝트 개요
- SKALA 부트캠프의 HTML/CSS/JavaScript 과제로 만드는 "개인 포털" 정적 사이트.
- 과제 원문은 프로젝트 루트의 html과제.txt / css과제.txt / javascript과제.txt 3개 파일에 있음 (처음엔 RTF로 대화에 붙여넣었다가, 나중에 사용자가 실제로 이 txt 파일들이 프로젝트 안에 있다고 알려줘서 다시 읽고 대조 검증했음).
- Git 상태: 사용자가 알아서 커밋/푸시하고 있음 (현재 브랜치 homework1, 최신 커밋 "homework13", working tree clean, origin과 동기화됨). 이 세션에서 나는 git add/commit을 한 적 없음 — 다른 컴퓨터에서는 그냥 git pull로 최신 상태를 받으면 됨.

## 2. 파일 구조
- html/index.html — 포털 허브 (가장 기능이 많은 파일)
- html/myProfile.html, myClass.html, myTrip.html, holiday.html — 개인 콘텐츠 페이지
- html/signUp.html, signUpResult.html, login.html — 회원가입/로그인 플로우
- css/style.css — 전체 스타일시트 (CSS 변수 기반 다크모드 포함)
- script/ — 기능별로 파일 분리 (아래 4번 참고)
- media/ — 여행 사진 3장(리장.jpg, 세부.jpg, 홋카이도.jpg), music.mp3, video.mp4
- 참고: 프로젝트 루트에 "scirpt"라는 오타 빈 폴더가 있는데 이건 사용자가 실수로 만든 걸 그대로 둔 것. 실제 스크립트는 전부 script/ 폴더에 있음. 헷갈리지 말 것.

## 3. 중요한 컨벤션 / 설계 결정
- 파일명: myProfile.html, myClass.html, myTrip.html, signUp.html, signUpResult.html은 과제 원문대로 my* 규칙. 단 홈일 페이지는 원래 있던 파일명 holiday.html을 그대로 유지하기로 사용자가 결정함 (myHoliday.html 아님). nav 링크도 실제 파일명(holiday.html)에 맞춰져 있음.
- 코드 스타일: <!doctype html> 소문자, 2-space 들여쓰기(head/body), 내부 콘텐츠는 4-space. 세미콜론 있는 JS, 한글 주석.
- 모든 HTML 파일의 <head>에 다음 부트스트랩 스크립트가 있음 (다크모드/글자크기가 페이지 로드 시 깜빡임 없이 바로 적용되도록):
  document.documentElement.dataset.theme = localStorage.getItem("theme") || "light";
  document.documentElement.style.fontSize = (Number(localStorage.getItem("fontScale")) || 1) * 100 + "%";
  → 새 페이지를 추가하면 이 스니펫도 같이 넣어야 다크모드/글자크기가 유지됨.
- 다크모드 기본값은 라이트모드 (시스템 설정 무시하고 항상 light로 시작, 사용자가 직접 토글해야 dark로 바뀜).
- CSS는 css/style.css 하나에 전부 있고, 과제1~6 섹션 주석(/* ===== 과제1: ... ===== */)으로 구간을 나눠놨음. CSS 변수(:root, [data-theme="dark"])로 라이트/다크 색상을 관리함.

## 4. script/ 폴더 파일별 역할
- darkMode.js — 다크모드 토글 (index.html 상단 버튼에서만 호출, 다른 페이지는 위 부트스트랩 스니펫으로만 적용)
- fontSize.js — 글자 크기 조절 (80%~140%, index.html 상단에 "가− 100% 가+" 그룹 버튼)
- clock.js — 실시간 시계 (index.html 사이드바)
- visitCount.js — localStorage 기반 "이 브라우저에서 몇 번째 방문" 카운터 (진짜 전체 방문자 수 아님, 정직하게 그렇게 표기함)
- auth.js — 로그인 시스템 전체 (서버 없음, localStorage를 계정 저장소처럼 사용). signUp.html 제출 시 계정 저장 → login.html에서 대조 로그인 → index.html 우측 상단에 "OOO님 환영합니다!" 표시. 로그인 여부와 기능 차이는 전혀 없음(순수 장식용).
- uiToggles.js — 자바스크립트 놀이터 섹션 접기/펼치기 (기본값: 펼쳐진 상태)
- upDown.js, grade.js, bag.js, rps.js, lotto.js, bmi.js — JS 놀이터의 미니게임/도구들. 원래 과제 스펙은 prompt()/alert() 사용을 요구했지만, 사용자 요청으로 전부 입력창+버튼 기반 DOM UI로 바꿈 (아래 5번 참고).
  - bag.js: 물건 추가/삭제 가능한 인터랙티브 가방 (원래는 그냥 보여주기만 하면 됐는데 확장함)
  - rps.js: 가위바위보, 연속 승리 횟수 + 최고 기록(localStorage 저장) 포함
- weatherAPI.js / realtimeInfo.js — 과제 스펙대로 모듈 분리(export/import). Open-Meteo API로 기온+습도+풍속 조회 (실시간 세계 날씨, 6개국 선택 가능)
- location.js — 브라우저 Geolocation API로 실시간 위치 조회 시도. **현재 임시로 비활성화 상태** (아래 6번 참고)
- encyclopedia.js — 위키백과 REST API 검색 (fetch/async-await)
- classSchedule.js — myClass.html 전용. 22주차 전체 커리큘럼 데이터를 갖고 있고, 각 주차의 "주요 학습 내용"을 클릭하면 그 주의 요일별(월~금) + 시간대별(09-12/점심/14-18) 상세 시간표를 팝업으로 보여줌
- tripSlider.js — myTrip.html 여행 사진 캐러셀 (한 장씩 슬라이드)

## 5. 원본 과제 스펙과 의도적으로 다르게 구현한 부분
(전부 사용자가 대화 중 명시적으로 요청해서 바뀐 것들. 마지막에 "다 괜찮은 것 같다"고 확인받음 — 즉 이대로 유지하기로 합의됨)
1. myClass.html — 원래 요구사항은 <table> 헤더가 "시간/요일"이고 2시간 이상 강의는 rowspan, 점심시간은 colspan. 지금 메인 표는 "월/주차/기간/내용" 구조(월단위 커리큘럼 요약)로 바뀌었고, 진짜 시간/요일+점심 병합 표는 주차를 클릭했을 때 뜨는 팝업에만 있음.
2. myTrip.html — CSS 과제(미션4)는 "Grid로 3열 배치"를 요구하는데, 지금은 캐러셀(슬라이더)로 한 장씩 넘기는 방식임.
3. 업다운 게임 / 성적 계산기 — 원래 prompt()/alert() 사용이 명시된 요구사항인데, 지금은 입력창+버튼 기반 UI로 바뀜.
4. index.html의 h1 타이틀 애니메이션 — CSS 과제는 "부드럽게 페이드인되는 등장 애니메이션"을 요구하는데, 실제 구현은 계속 반짝이는(glow) 무한 반복 애니메이션이고 모든 페이지의 h1에 적용돼 있음(원래는 index.html만 해당). 이건 내가 예전에 애매한 번역본을 잘못 이해해서 생긴 미스인데, 사용자에게 보고했더니 "다 괜찮은 것 같다"며 그대로 두기로 함. 즉 의도적 deviation은 아니고 실수인데 사용자가 그냥 넘어가기로 한 것 — 혹시 나중에 문제 되면 css/style.css의 @keyframes glow 부분을 fadeIn으로 바꾸면 됨.

## 6. 알려진 이슈 / 임시 처리 중인 것
- **위치 정보(Geolocation)**: 사용자 macOS에서 Wi-Fi 켜져 있고, macOS 위치서비스도 켜져 있고, Chrome 권한도 허용했는데도 계속 POSITION_UNAVAILABLE 에러 발생. 심지어 외부 사이트(browserleaks.com/geo)에서도 실패해서 이 프로젝트 코드 문제가 아니라 기기/OS 레벨 문제로 결론남 (macOS 설정 > 개인정보 보호 및 보안 > 위치 서비스 > 시스템 서비스 > Wi-Fi 네트워킹이 꺼져있을 가능성을 안내했음, 확인 결과는 아직 못 받음).
  → script/location.js 안에 실제 geolocation 코드는 그대로 남겨뒀고, 함수 맨 위에서 무조건 "🇰🇷 대한민국" 고정 텍스트를 표시하고 return 하도록 임시 처리해놨음. 나중에 위치 서비스 문제 해결되면 그 이른 return 두 줄만 지우면 원래 로직이 다시 동작함.

## 8. 작업 스타일 관련 참고
- 사용자는 "제안해줘" 류의 질문을 자주 하고, 그럴 땐 먼저 옵션 몇 개를 제시하고 사용자가 고른 것만 구현하는 흐름을 선호함 (예: 놀이터 추가 기능 8~10개 제안 → 사용자가 2개 골라서 그것만 구현).
- 버그 리포트를 주면 원인을 구체적으로 설명해준 뒤 고치는 걸 좋아함 (예: rowspan이 다음 줄까지만 덮는 문제, box-sizing 문제, [hidden] 속성보다 ID 선택자가 우선순위 높아서 생긴 문제 등을 매번 원인 설명 + 수정 순서로 진행함).
- 기능 추가할 때마다 index.html의 "최근 소식/업데이트" 목록에도 반영해달라고 요청하는 패턴이 있음 — 새 기능을 추가하면 이 목록도 같이 업데이트해줄 것.
